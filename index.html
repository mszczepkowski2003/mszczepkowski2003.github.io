Flying Object Classification with MobileNetV3

Tech Stack: Python, TensorFlow, MobileNetV3-Large, Docker, FastAPI
Main Language: English

  • Key challenges:
    - Data Scarcity & Splitting: Managed a limited dataset by implementing a 70/15/15 (Train/Val/Test) split to ensure robust validation and final testing despite small sample sizes.
    - Preventing Overfitting: Addressed the risk of high variance by implementing various Data Augmentation strategies, effectively expanding the training set and improving the model's ability to generalize to new environments.
    - Hardware Constraints: Balanced high-performance requirements with the limited CPU power of potential deployment hardwe (e.g. CCTV cameras). This required selecting an architecture that prioritizes low computational latency without sacrificing accuracy.
  • Data Engineering & Optimization:
    - Pipeline Efficiency: Utilized TensorFlow's .cache() and prefetching mechanisms to maximize training throughput, ensuring the GPU remained saturated and preventing I/O memory bottlenecks.
    - Preprocessing: Engineered a pipeline that integrated real-time data augmentation.
  • Architecture & Transfer Learning Strategy:
    - Architecture selection: Selected MobileNetV3 Large for its use of depthwise separable convolutions, making it ideal for edge devices with restricted power and memory capacity
    - Two-stage training:
    • Feature extraction: Initially froze the MobileNet backbone (pretrained on imageNet) to train a custom classification head.
    • Fine-tuning: Unfroze deep layers and retrained with a significantly lower learning rate to align the pretrained weights with specific visual features of drones, birds and aircrafts.
  • Results & Insights:
    - Performance: Achieved a 95% weighted accuracy, demonstrating strong ability of the model to separate classes even with constraints of small fine-tunning dataset.
    - Error Analysis: Performed post-training error analysis to identify specific edge cases - such as distant or blurry objects that proved problematic for the model, providing a roadmap for future data collection.
  • Deployment:
    - Production Readiness: Deployed the final model as a RESTful API using FastAPI, providing a lightweight interface for real-time inference.
    - Containerization: Fully dockerized the application to ensure environmental consistency and easy portability across different production servers.
